Python → PHP

PHP for Python Developers

A practical translation guide — read and write PHP by mapping it to what you already know.

You already understand variables, functions, objects, and control flow. PHP speaks the same concepts with a different syntax. This guide skips the beginner tutorials and goes straight to the translation layer.

At a glance

Keep this table handy for the most common syntax mappings.

Concept Python PHP
Variable x = 1 $x = 1;
Output print(x) echo $x;
String concat "a" + "b" "a" . "b"
Null None null
Boolean True / False true / false
List / Array [1, 2, 3] [1, 2, 3]
Dict / Assoc {"a": 1} ["a" => 1]
Function def f(): function f() {}
Class class Foo: class Foo {}
Ternary x if c else y $c ? $x : $y
String interp f"hi {name}" "hi {$name}"
Equality == == (loose) / === (strict)

Variables & types

Both languages are dynamically typed. The only real difference in day-to-day variable use: PHP requires a $ prefix on every variable reference.

Python 
# Python: dynamic typing
name = "Alice"
count = 42
items = [1, 2, 3]

def greet(person):
    return f"Hello, {person}"
PHP 
// PHP: same concept, $ prefix
$name = "Alice";
$count = 42;
$items = [1, 2, 3];

function greet($person) {
    return "Hello, {$person}";
}
Note PHP variables use $ on both assignment and access — everywhere. Function names and class names do not use $. Semicolons are required at the end of every statement.

Collections

PHP's array is a hybrid of Python's list and dict. The same [] syntax produces both, distinguished by whether you provide keys.

Indexed arrays (like Python lists)

Python 
fruits = ["apple", "banana"]
fruits.append("cherry")
len(fruits)  # 3
fruits[0]      # "apple"
PHP 
$fruits = ["apple", "banana"];
$fruits[] = "cherry";
count($fruits);  // 3
$fruits[0];       // "apple"

Associative arrays (like Python dicts)

Python 
user = {
    "name": "Alice",
    "role": "admin"
}
user["name"]  # "Alice"
PHP 
$user = [
    "name" => "Alice",
    "role" => "admin"
];
$user["name"];  // "Alice"
Watch out PHP uses => (fat arrow) not : for key-value pairs. The same [] reads and writes both indexed and associative arrays. Use array_key_exists() or isset() instead of in or has_key().

Functions

Python's def becomes PHP's function. Parameters get $ in their declaration. Return type hints go after a colon.

Python 
def add(a, b):
    return a + b

# Lambda
square = lambda x: x ** 2
PHP 
function add($a, $b): int {
    return $a + $b;
}

// Arrow fn (PHP 7.4+)
$square = fn($x) => $x ** 2;
Note PHP functions have their own variable scope — a function cannot see a variable from the caller unless you declare it global or pass it as a parameter. Arrow functions automatically capture outer scope (like Python closures).

Control flow

The keywords map almost one-to-one. Just remember braces instead of colons+indentation, and elseif is one word.

Python 
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "C"

for item in items:
    print(item)
PHP 
if ($score >= 90) {
    $grade = "A";
} elseif ($score >= 80) {
    $grade = "B";
} else {
    $grade = "C";
}

foreach ($items as $item) {
    echo $item;
}
Gotcha PHP's foreach operates on a copy of the array by default. Use foreach ($items as &$item) to mutate in place — and unset($item) after to avoid reference leaks.

Classes & objects

PHP 8+ supports constructor property promotion that will feel familiar if you've used Python dataclasses. Visibility is explicit (public, protected, private) and typed properties are standard.

Python 
class Dog:
    def __init__(self, name):
        self.name = name

    def bark(self):
        return f"{self.name} says woof!"
PHP 
class Dog {
    public function __construct(
        private string $name
    ) {}

    public function bark(): string {
        return "{$this->name} says woof!";
    }
}
Note Access properties with $this->name not self.name. PHP uses -> (arrow) for property and method access on objects, :: for static access.

PHP quirks that trip up Python devs

  • The $ is mandatory. Forget it and you get a notice or an undefined constant error. Train your muscle memory early.
  • Semicolons are required. PHP only infers statement endings at ; — missing one breaks the parse.
  • == lies. PHP's == coerces types ("1" == 1 is true). Use === for strict equality that checks type too.
  • Concatenation uses . not +. The + operator does numeric addition (and will coerce strings to numbers).
  • Arrays are ordered maps. A PHP array is always ordered by insertion. array_merge() replaces + on dicts, array_push() / $arr[] = replaces list.append().
  • Scope is function-level. Variables from the outer scope are not visible inside a function unless explicitly imported with use (closures) or global.
  • Methods are not automatically references. Objects are passed by handle (like Python), but arrays and scalars are passed by value. Use & for explicit references.
